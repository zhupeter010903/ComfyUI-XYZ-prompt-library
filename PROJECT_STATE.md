## 4. Active Assumptions

> 这些是当前已经写进文档 / 代码、并被后续任务**默认依赖**的设计假设。任何变更都需同步回 `PROJECT_SPEC.md` / `ARCHITECTURE.md`。

1. **SQLite 是唯一事实来源（C-1）**。所有 UI 查询命中索引，文件系统只在 `indexer` / `thumbs` / `metadata` / 二进制端点内被访问。PNG 块只是衍生镜像。**【T07 加固】**：`UpsertImageOp.apply` 在 UPSERT 时对 `favorite` / `tags_csv` 用 `COALESCE(excluded.x, image.x)`——重扫一张不含 `xyz_gallery.*` 块的 PNG **永远不会**用 NULL 盖掉 DB 已有值。**【T08 进一步加固】**：`thumbnail_cache` 是 on-disk `.webp` 的**权威源**；T08 `thumbs.request` "先物理后入队" 保证 DB "少记不多记"；反向的孤儿 `.webp` 由 T26 每日 janitor 回收，是预期漂移窗口而非 bug。**【T10 复核】**：HTTP 只读路径 **`/image/{id}` / `/image/{id}/neighbors` / `/image/{id}/workflow.json`** 均以 DB 的 `image.workflow_present` 做"有无 workflow" 的权威判据——即使磁盘 PNG 真的有 `workflow` tEXt 块，只要 DB 说没有就 404（§4 #23 语义在 HTTP 端被坚决执行），**严禁**未来任何任务在 routes 层做 "DB 说没有就去磁盘兜底扫一下" 的退化。
2. **写入路径单一化（ARCHITECTURE §4.6）**。除迁移脚本外，所有 SQLite 写都经 `repo.WriteQueue` 单一写入者线程串行。**【T04 / T05 / T07 / T08 / T17 已代码化 + 实测】**。**【T10 复核】**：`routes.py` 全模块**零** `sqlite3` import、**零** `cursor.execute`；唯一接触 DB 的方式是 `_run(repo.get_image / repo.list_images / repo.folder_tree / repo.neighbors / folders.list_roots, …)` 四条读路径 + `_run(thumbs.request, …)` 间接走 `db.connect_read` 单行 `SELECT`；写路径经 `thumbs.request` → `enqueue_write(LOW, InsertThumbCacheOp / _TouchFlushOp)` **以及** `indexer` → `enqueue_write(LOW, UpsertImageOp)`（T15 词表链接同事务落在 `UpsertImageOp.apply` 内）**以及** `metadata_sync` → `enqueue_write(LOW, SetSyncStatusOp / SetSyncFailedOp / SetSyncHardFailedOp)`，单写者不变式在"HTTP 热路径 + 1 s 内多次同 id 刷屏"的新压力模式下仍稳定（生产 QA 实测 id=3833 多次 GET 无 `database is locked`）。
3. **冷启动不全量重扫（C-3 + NFR-8）**。`(size, mtime_ns)` 指纹短路；首次扫描单次性，后续启动只跑 delta scan。**【T07 已代码化】**。
4. **缩略图按需生成、永不批量预生成（NFR-7 / §8.3）**。WebP 320×320 cover、`sha1(path+mtime_ns)` 寻址、2 字符分片目录、`thumbnail_cache` 表为权威 LRU。**【T08 已代码化】**。**【T10 端到端贯通】**：`/xyz/gallery/thumb/{id}` 是唯一生产触发入口，生产 QA 实测 id=3833 首次 GET 触发 `_generate_and_save` 生成 11226 B WebP + `InsertThumbCacheOp(LOW)` 落库；第二次 GET 纯磁盘命中 + `touch()` 进 `_touch_set`；`?v=<mtime_ns>` URL 与 `Cache-Control: public, max-age=31536000, immutable` 形成端到端的"mtime 变 → hash_key 变 → .webp 路径变 → 浏览器 cache miss" 闭环。LIFO / LRU / janitor 仍属 **T26**，T10 模块 docstring 零"T26 锚点"残留。
